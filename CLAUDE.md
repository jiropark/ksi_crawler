# KIS 실시간 스크리너 + 매매 전략 알고리즘

## 프로젝트 개요
- **API**: 한국투자증권 (KIS Open API) - REST + WebSocket
- **실행환경**: Linux (NAS/서버), Python 3.10+
- **출력**: 터미널 출력 + CSV 저장 + SQLite
- **스케줄**: 장 중 09:00~15:30, 매 1분 실행

---

## 1. 기본 스크리닝 조건 (전 전략 공통)

| # | 조건 | 기준값 |
|---|------|--------|
| ① | 당일 누적 거래대금 | ≥ 1,000억 |
| ② | 전일 대비 등락률 | ≥ +10% |
| ③ | 외인 + 기관 동시 순매수 | 당일 기준 |
| ④ | 시가총액 | ≥ 1,000억 (잡주 제외) |
| ⑤ | 거래량 전일 대비 | ≥ 3배 |
| ⑥ | 등락률 상한 | < 27% (상한가 근처 제외) |
| ⑦ | 신규 상장 | 10일 이후만 (IPO 제외) |
| ⑧ | 고가 대비 현재가 | -5% 이내 |

## 2. 전략별 매수 알고리즘

---

### 🔵 전략 A. 스캘핑 (Scalping)
> 1~5분 단위 빠른 진입/청산, 소수익 반복

#### 매수 조건
```
① 1분봉 거래대금 ≥ 50억
② 체결강도 > 120
③ 호가창 매수잔량 > 매도잔량 × 2배
④ 직전 2분봉 연속 양봉
⑤ 전고점 -1% 이내 접근
```

#### 홀딩 / 매도 조건
```
✅ HOLD  : 체결강도 > 100 유지
           + 프로그램 순매도 < 10억

⚠️ 경고  : 거래대금 급감
           OR 체결강도 100 이하 꺾임

🔴 SELL  : 프로그램 순매도 ≥ 10억 연속 2분
           OR 음봉 전환
           OR 진입가 -1% 이탈
```

#### 익절 / 손절
```
익절 : +1~2% 즉시
손절 : -0.5~1% 즉시
최대 보유시간 : 10분
```

---

### 🟡 전략 B. 눌림목 (Pullback)
> 급등 후 횡보/눌림 → 지지 확인 → 재폭발 시 진입

#### 매수 조건
```
① 당일 이미 +5~15% 급등한 종목 (기본 스크리닝 통과)

[횡보 감지]
② 직전 3분↑ 횡보 (고가-저가 변동폭 ≤ 1.5%)
③ 횡보 구간 1분 평균 거래대금 ≤ 10억 (매도세 소진)

[재폭발 감지]
④ 현재 1분 거래대금 ≥ 50억 재돌파
⑤ 횡보 평균 거래대금 대비 5배↑ 터짐
⑥ 체결강도 > 110

[방향성 확인]
⑦ 직전 3분봉 연속 양봉 OR 저점 상승 구조
⑧ 전고점(횡보 전 장중 최고가) -2% 이내 접근
⑨ 외인/기관 순매수 유지 중
```

#### 홀딩 / 매도 조건
```
✅ HOLD  : 프로그램 순매도 < 10억
           + 체결강도 > 100
           + 분위기 유지 (양봉 기조)

⚠️ 경고  : 프로그램 순매도 ≥ 10억 1회 발생
           → 1분 관망, 주가 반응 확인
           → 주가 버티면 홀딩 / 밀리면 즉시 매도

🔴 SELL  : 프로그램 순매도 ≥ 10억 연속 2분
           OR 프로그램 매도 + 음봉 + 주가 하락 전환
           OR 눌림 저점 이탈
```

> 💡 프로그램 매도 나와도 주가 안 밀리면 → 오히려 강한 신호 (홀딩)
> 💡 프로그램 매도 + 주가 같이 밀리면 → 세력 이탈 신호 (즉시 매도)

#### 익절 / 손절
```
1차 익절 : 전고점 +2~3% 돌파 후 50% 청산
2차 익절 : 신고가 갱신 시 트레일링 (+1% 단위)
손절     : 눌림 저점 이탈 (-2~3%)
```

---

### 🟠 전략 C. 종가배팅 (End-of-Day)
> 장 마감 30분 전 강한 종목 홀딩 → 다음날 갭상승 노림

#### 매수 조건 (실행 시간: 14:30~15:00)
```
① 기본 스크리닝 조건 전부 통과
② 외인/기관 당일 지속 순매수
③ 장 막판 프로그램 매수 유입 (순매도 X)
④ 종가 전고점 부근 OR 신고가 근처 마감 예상
⑤ 당일 고가 대비 -3% 이내 마감 예상
⑥ 52주 신고가 돌파 OR 근접
⑦ 뉴스/테마 모멘텀 유지 중
```

#### 다음날 홀딩 / 매도 조건
```
✅ HOLD  : 갭상승 시가 + 추가 상승 분위기 유지
           + 외인/기관 매수 지속

⚠️ 경고  : 갭상승 후 시초가 밀림 시작

🔴 SELL  : 갭하락 시가 → 즉시 손절
           OR 시초가 -2% 이탈
           OR 외인/기관 다음날 매도 전환
```

#### 익절 / 손절
```
목표     : 다음날 +3~5% 갭상승
1차 익절 : 시초가 +3%↑ → 50% 청산
손절     : 당일 종가 기준 -2% 이탈 (다음날 시초)
```

---

### 🟣 전략 D. 모멘텀 스윙 (Swing)
> 2~3일 추세 추종, 눌림목 전략의 상위 호환

#### 매수 조건
```
① 눌림목(전략 B) 조건 전부 충족
② 외인/기관 2일↑ 연속 순매수
③ 52주 신고가 돌파
④ 거래대금 3일 연속 증가
⑤ 섹터/테마 모멘텀 지속 중
```

#### 익절 / 손절
```
익절 : +5~10% 분할 청산 (3회 분할)
손절 : 5일선 이탈
```

## 5. 시스템 구성

```
kis_screener/
├── CLAUDE.md              ← 이 파일 (전략 정의)
├── config.py              ← API 키, 계좌번호, 파라미터 설정
├── auth.py                ← 토큰 발급/갱신/캐싱
├── api/
│   ├── rest.py            ← REST API 호출 (재시도 포함)
│   └── websocket.py       ← 실시간 체결 WebSocket
├── screener/
│   ├── filters.py         ← 기본 스크리닝 조건 ①~⑧
│   └── patterns.py        ← 전략별 패턴 감지 (A/B/C/D)
├── strategy/
│   ├── scalping.py        ← 전략 A
│   ├── pullback.py        ← 전략 B (눌림목)
│   ├── eod.py             ← 전략 C (종가배팅)
│   └── swing.py           ← 전략 D
├── signal/
│   ├── hold.py            ← 홀딩 판단 로직
│   └── sell.py            ← 매도 판단 로직
├── storage/
│   ├── db.py              ← SQLite 저장
│   └── csv_writer.py      ← CSV 일별 저장
├── notify.py              ← 텔레그램 알림 (선택)
├── scheduler.py           ← APScheduler, 장시간 체크
└── main.py                ← 진입점
```

---

## 6. 개발 우선순위

```
Phase 1 (핵심)
  - auth.py : 토큰 자동 갱신
  - rest.py : 재시도 + Rate Limit 제어
  - filters.py : 기본 스크리닝 조건 ①~⑧
  - patterns.py : 눌림목 패턴 (전략 B 우선)
  - scheduler.py : APScheduler + 장시간 체크

Phase 2 (확장)
  - websocket.py : 실시간 체결강도/호가
  - hold.py / sell.py : 홀딩/매도 판단
  - notify.py : 텔레그램 알림

Phase 3 (고도화)
  - 전략 A (스캘핑) 자동화
  - 전략 C (종가배팅) 자동화
  - 백테스팅 모듈
```

## 7. 환경 설정 예시 (config.py)

```python
# KIS API
APP_KEY    = "YOUR_APP_KEY"
APP_SECRET = "YOUR_APP_SECRET"
ACCOUNT_NO = "YOUR_ACCOUNT_NO"
IS_REAL    = False  # True: 실전, False: 모의투자

# 스크리닝 기준값
MIN_TRADE_AMOUNT   = 100_000_000_000  # 1000억
MIN_CHANGE_RATE    = 10.0             # +10%
MIN_1MIN_AMOUNT    = 5_000_000_000   # 50억
MAX_CHANGE_RATE    = 27.0            # 상한가 근처 제외
MIN_MARKET_CAP     = 100_000_000_000 # 시총 1000억
MIN_VOLUME_RATIO   = 3.0             # 전일 대비 거래량 3배

# 눌림목 패턴
SIDEWAYS_MINUTES       = 3     # 횡보 최소 지속 분
SIDEWAYS_VOLATILITY    = 1.5   # 횡보 변동폭 %
SIDEWAYS_MAX_AMOUNT    = 10_000_000_000  # 횡보 중 최대 거래대금 10억
EXPLOSION_RATIO        = 5.0   # 횡보 대비 거래대금 폭발 배수
NEAR_HIGH_THRESHOLD    = 2.0   # 전고점 -2% 이내

# 홀딩/매도
PROGRAM_SELL_WARNING   = 10_000_000_000  # 프로그램 매도 경고 10억
PROGRAM_SELL_CONSEC    = 2      # 연속 n분 → 즉시 매도
CHEGYOL_STRENGTH_MIN   = 100   # 체결강도 최소값

# 스케줄
MARKET_OPEN  = "09:00"
MARKET_CLOSE = "15:30"
EOD_START    = "14:30"
SCAN_INTERVAL_SEC = 60
```
